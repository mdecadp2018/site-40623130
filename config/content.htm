除錯

資料在合併時有時可能會出現錯誤,這在協同者間很容易發現,以下網址將教你如何遇到衝突:

Here

經過文章內的步驟一步一步做下去能發現,資料沒有發生衝突也沒合併,那是因為資料沒有被寫入支內部或是說沒有切換(製作)分支就存取資料,所以在Head的情況下

如何合併呢?

只要再新增一個分支並將資料寫入,以ts為新分支

並合併這三個分支,很幸運的(?)成功測到衝突了

改一下衝突後,在上推資料,就成功處理完了

Instruction

只舉常用的指令

Cmd

dir (查看目錄)

help (指令求救)

python +名(利用python執行"+名"檔案)

cls (清除指令紀錄)

cd +名 (移動)

檔名開頭+TAB鍵 (列出全名-可連續按)

attrib -r -s -h X:\. /s /d (將X槽的檔案從隱藏還原)

Git

git add (存取資料)

git commit -m "name" (命名此批欲上傳資料)

git pull (更新至近端資料)

git push (上傳至遠端)

git version (查看git版本)

git update (更新github版本)

git revert -m "name"(還原commit -m name紀錄)

git cleav -n -f(-n列出欲清除的資料 -f真的清除)

git status (狀態查詢)

git branch +名 (新建分支)

git checkout +名 (切換分支)

 git merge +名 (合併分支)

git log (查看檔案版本)

git log --oneline --graph --all (檢查各版本間的關聯與樹狀圖)

F鍵 : 切換視野至草圖平面

N鍵 : 正視圖面

M鍵 : 切割

L鍵 : 草圖畫直線

C鍵 : 草圖畫圓
